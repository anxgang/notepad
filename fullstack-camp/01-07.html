# 7-上傳專案到 Heroku

預計學習時間: 1.5小時

# 7-1 註冊 Heroku 帳號

# 目的

讓網站能夠公開給大家看。

# 1. 首先先註冊一個 Heroku 帳號

https://www.heroku.com

# 2. 安裝 Heroku Command Line (Heroku Toolbelt)

Heroku 所推出的環境命列工具包。
前往 https://toolbelt.heroku.com 依照網站指示下載安裝。

檢查是否完成安裝，可以輸入 heroku --version
應該會看到：

（版本可能會不同，這是正常的）

# 3. 完成 Heroku 的 SSH-key 的設定

先輸入 heroku login 確認登入，輸入email、密碼（密碼不會顯示，輸入完直接按return即可）

再輸入 heroku keys:add 即可

# 部署（deploy）是什麼？

我們所有的程式設計工作都是在自己的電腦完成，所以只能在自己的瀏覽器上看到成果，如果想要讓網站公開給大家看，但就必須將它放到伺服器上，這個把做好的網站安放到伺服器上的動作，就叫部署（deploy）。

# 什麼是 Heroku ？

Heroku 是 Rails 社群中，最方便穩定的網站部署工具。如果你開發了小型的網站，Heroku 提供了非常便利的部署工具。（用 Git 上傳就能部署）。

# 7-2 上傳專案到 Heroku

# 目的

上傳專案到 Heroku

# 第一步 - 建立 heroku 專案

在終端機（Terminal）裡輸入 heroku create

heroku create 會在 Heroku 的系統裡面註冊新的應用程式。跑完之後你應該會看到輸出裡面告訴你新的應用程式的 URL。

注意

請不要重複執行 heroku create 這個指令
每輸入一次 heroku create 都會產生一個新的 herokuapp，免費帳號預設上限為5個。
超過 5 個的時候，請手動登入自己的 heroku 網址刪除app。

# 第二步 - 調整 Gemfile

在終端機（Terminal）輸入 code .

打開 Gemfile 文件，進行如下修改：

# 1.遷移 sqlite3

把 gem 'sqlite3', '~>1.4' 整個剪下，貼到 group :develop, :test do ... end 之中

變成下圖這樣

# 2.加 'pg' 這個 gem

並且在剛剛 gem 'sqlite3 的下方，再補上以下程式碼

# gemfile
group :production do 
  gem 'pg'
end

如圖所示：

請記得存檔！

# 3. 執行 bundle install

調整完 gemfile 後記得都要重新輸入 bundle

# 4. 調整 database.yml

因為 heroku 必須使用 postgresql 所以將 production 的資料庫其調整為以下程式碼

# config/database.yml
production:
  <<: *default
  adapter: postgresql
  encoding: unicode
  pool: 5

# 第三步 - 儲存目前版本

分別在終端機輸入以下程式碼

git add .
git commit -m "move sqlite3 to dev group & add pg to production group"

# 第四步 - 推上 heroku

找到 config/routes.rb 檔案。
在第 2 行加入 root "topics#index"

讓首頁根目錄指到 topics 的 index 清單頁。

之後我們一樣在終端機（Terminal）執行：

git add .
git commit -m "add root path"

最後再輸入 git push heroku master

（部署中...會等一段時間）

看到 remote: Verfying deploy... done. 就代表部署成功

# 第五步 - 建立 heroku 資料庫

部署成功後，在終端機（Terminal）輸入

heroku addons:create heroku-postgresql:hobby-dev 新增一個資料庫

再輸入

heroku run rake db:migrate 建立資料表

# 第六步 - 查看網站

最後在終端輸入 heroku open 就可以打開剛剛部署到 heroku 上的 app 了。恭喜你成功完成了部署。

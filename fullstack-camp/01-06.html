# 6-體驗 Git

預計學習時間: 1.5小時

# 6-1 Git 基本指令

# 目標

學會幾個 git 的基本指令：

  • git config
  • git init
  • git add
  • git status

在這章我們只會講解必要的指令。

# 步驟

# 1. git config

就像是首次玩遊戲一樣，你必須設定遊戲玩家角色叫什麼。這裡我們必須要設定兩個參數：

git config --global user.name "你的名字"

注意 "你的名字"要修改成自己的名字

git config --global user.email "你的email@xxx.xxx"

注意 "你的email@xxx.xxx" 要修改成自己用來註冊github的那個email

這樣以後進行存檔時，就會紀錄你是這次進度的存檔者。

# 2. git init

注意

Rails 5.0+ 以後的版本都會自動 git init 所以可略過

請注意，一定要在你之前操作過的 first_app 子目錄裡輸入以下命令。如果你在根目錄裡輸入了git init，將會大幅降低整個系統的執行速度。

有關判斷是否在該子目錄的辦法，請點擊說明文檔

在 first_app 子目錄下，輸入git init

初次設定進度儲存庫，也就是開始使用 git 來存檔。

# 3.git add & git status

輸入 git add README.md，再輸入 git status

你會發現 README.md 變成綠色，其餘的檔案還是紅色。
這表示我們已經預備將 README.md 設定為預備儲存進度的檔案。

# 4.git commit

輸入 git commit -m "add README" 真正將這個檔案存進去進度裡。

接著我們再輸入一遍 git status，發現還是有很多檔案，沒有被存進去進度區裡面。

所以這次我們也要將剩下的檔案也存進去進度裡面。

輸入 git add . ，這樣就可以把這個目錄夾下“所有還沒有被加入 git 歷史資料庫的檔案”設為“預備儲存”的狀態。然後輸入 git commit -m "add rest files" 儲存剩下所有的檔案。

# What is Git?

Git 是 Linux 之父 Linus Torvalds 開發出來的一套程式碼版本控制系統。你可以把 git 想像成遊戲時存檔的工具。每當程式開發到一定的段落，就使用 git 存檔一下。你可以把 Git 理解成就是小叮噹的時光機。

# 6-2 上傳 Github - (1) 設定使用者名稱

# Github

Github 是 Opensource 社群中，重要的資源集散地。幾乎人人都有一個帳號，幾乎是強制必備。請在此註冊一個帳號。

http://github.com

# 在 Github 上設定自己的電子簽名

你可以這樣想像，每一台筆電都有自己的電子簽名，他就是你的 SSH Key。而 SSH Public Key 你可以想像是在銀行( Github）留的印鑑卡，去銀行辦事時，跟你的私有 Key 一對，就能確定這是你本人。

所以在這裡我們要進行一個在 Github 留印鑑卡的動作

# 步驟 1

在 Terminal 先輸入 cat ~/.ssh/id_rsa.pub 看看有沒有東西。

  1. 有的話 copy 整個內容，到步驟 3。
  2. 沒有的話到步驟 2。

# 步驟 2

如果沒有 id_rsa.pub 這個檔案, 請輸入 ssh-keygen 連按三個 enter

然後再輸入一次 cat ~/.ssh/id_rsa.pub , 把出來的內容 copy 起來

# 步驟 3

前往 https://github.com/settings/ssh

點選 New SSH key 建立一組新的 SSH key

Title 可以隨意輸入，取個好辨識的名稱即可，例如： 'MBPR 15'

Key 就貼入剛剛複製的 SSH key

完成後即可看到剛剛建立的項目

# 6-4 持續開發新功能並且上傳到 Github

以後要如何持續開發新功能，並且將開發成果上傳到 Github 呢?

其實在這個階段，你只要持續做 3 件事，就行了。

  • git add .（所有已異動的檔案，都加入新版本的範圍）
  • git commit -m "這次存檔的訊息" (建立新版本)
  • git push origin master （上傳到 github）

就可以持續把進度同步到 Github。

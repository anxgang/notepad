# 1-教材介紹

預計學習時間: 10分鐘

# 1-0 由此開始：學習說明

# 課程提醒

請務必在正式開課前完成課前作業，即： 《Rails 第一課》、《Rails 第二課》、《Rails 第三課》的作業。

# 1-1 前言

# 目標

為了讓你用最快的速度上手 Ruby on Rails，我們會使用“真實世界”的範例。

現在我們要實際做出一個投票系統， 這是一個基本可用的程式，讓使用者可以：

  • 檢視主題，用投票數來排序
  • 對主題投票
  • 開新主題、編輯主題、刪除主題
  • 貼張圖讓你看看它會長怎樣：

(這份教材是基於 RailsBridge 工作坊的 Intro to Rails改編）

# 練功目標

將初級作業完成之後，你會學會：

  • 基本 Ruby 語法
  • 要怎麼試一些 Ruby 程式碼 (irb)
  • 要怎麼把需求變成會動的 Rails 應用程式
  • 要怎麼把你的應用程式放到網路上
  • Rails 工程師會使用的基本工具（版本控制、編輯器、console、本機 server）

# 時程

  • 約 1 小時 Ruby
  • 約 4-6小時 Rails。

# 需求

我們會用這些來做：

  • ruby 2.7.1 用 RVM 安裝（Mac 或 Linux）
  • rails 6.0.3.2
  • bundler 2.0.2
  • VSCode 編輯器

這些應該都要在前一份教材就裝好，確定它們都沒問題。

# 如何確認一切沒問題？

執行終端機，輸入：

irb

2.7.1 :001 > 1 + 2
=> 3

2.7.1 :002 > require "active_support"
=> true

2.7.1 :003 > exit

這些都可以成功執行的話，就是沒問題了。

提示：在 irb 裡執行命令的最後記得輸入 exit 退出。

# 效能！效率！

我非常推薦你這樣做：

  • 打開終端機並開兩個標籤頁：

    • 一個放著跑 rails s

    • 一個用來做平常的終端機指令

      （有時會用來使用 irb —— 即 rails console，我們之後會提到，用過後記得輸入exit退出即可）

  • 重新打開你的瀏覽器，或是把其他視窗關掉

    • 打開新的視窗並開兩個標籤頁
    • 一個用來看本教學
    • 一個用來測試你的程式

  • 打開 VSCode 編輯器，然後不要關掉它。我們不會中途離開。

  • 把其他應用程式都隱藏起來，關掉 Twitter 、即時通訊程式、別的干擾物。

  • 把跟你互動的東西最少化，可以減少你分心在這些東西的次數與耗費的時間（比方說在瀏覽器裡打開 50 個分頁，這會干擾到你並且浪費你的時間）

# 格式

每一課的格式都會長這樣：

  • 目標

敘述我們要做什麼。

  • 步驟

跟著做，但是這時候你還不知道原理。

  • 解釋

關於這些步驟實際上做了什麼的細節，詳述原因與效果。將一切都連在一起。

# 注意

如果你剛剛完成第一課的環境建置

  1. 請在終端機執行 rails server 的視窗（或者標籤頁）中按 ctrl + c 終止執行，然後按 command + q 退出終端機。
  2. 請在 VSCode 編輯器的界面按 command + w 關閉所有的標籤頁，然後點擊狀態列左上角 Code > Quit 退出編輯器。

# 1-1 前言

# 目標

這一份是寫給 Rails 新手的中級訓練教材。
接下來會以一個“討論版”為主題，實際讓同學瞭解如何“從零打造”手寫出一個網站。

這個討論版會有：

  • 使用者註冊以及登錄功能
  • 開新討論區、留言功能
  • 加入討論區、退出討論區功能
  • 權限管理功能
  • 熱門文章排序功能
  • 瞭解什麼是 RESTful

當你把這次應用程式實作完成，並上線後，你會學會：

  • 怎麼把需求變成會動的 Rails 應用程式

  • 如何在 Rails 套版

  • 如何製作使用者登錄功能

  • 重要 Rails 觀念以及 API

    • CRUD
    • RESTful
    • member, collection
    • helper
    • partial
    • scope

  • 找到 Rails 第三方 gem 的技巧

  • 要怎麼把你的應用程式放到網絡上

這幾乎是一個 Rails 新手應該學的一切了，有了這些基礎技巧之後，你再學更難的技巧，或者是要自己做出複雜的東西，就不會那麼容易失敗了。

# 時程

  • 約 30 小時 Rails 開發
  • 建議每天使用 4 小時來完成作業（約需5-7天）
  • 重複練習三遍（相信我，你會越練越快的...）

# 需求

我們會用這些來做：

  • ruby 2.7.1 用 RVM 安裝（Mac 或 Linux）
  • rails 6.0.3.2
  • bundler
  • VSCode 編輯器

這些應該都要在第一份教材就裝好，確定它們都沒問題。

# 如何確認一切沒問題？

執行終端機，輸入：

irb

2.7.1 :001 > 1 + 2
=> 3

2.7.1 :002 > require "active_support"
=> true

2.7.1 :003 > exit

這些都可以成功執行的話，就是沒問題了。

提示：在 irb 裡執行命令的話，最後記得輸入 exit 退出。

# 效能！效率！

我非常推薦你這樣做：

  • 打開終端機並開兩個標籤頁：

    • 一個放著跑 rails s
    • 一個用來做平常的終端機指令

  • 重新打開你的瀏覽器（推薦chrome），或是把其他視窗關掉

    • 打開新的視窗並開兩個標籤頁
    • 一個用來看本教學
    • 一個用來測試你的程式

  • 打開 VSCode 編輯器，然後不要關掉它。我們不會中途離開。

  • 把其他應用程式都隱藏起來，關掉facebook 、微信和其他即時通訊軟件、別的干擾物。

  • 把跟你互動的東西最少化，可以減少你分心在這些東西的次數與耗費的時間（比方說在瀏覽器裡打開 50 個分頁，這會干擾到你並且浪費你的時間）

# 格式

每一課的格式都會長這樣：

  • 目標

敘述我們要做什麼。

  • 步驟

跟著做，但是這時候你還不知道原理。

  • 解釋

關於這些步驟實際上做了什麼的細節，詳述原因與效果。將一切都連在一起。

# 注意

如果你剛剛完成第二課的初級練習

  1. 請在終端機執行 rails server 的視窗（或者標籤頁）中按 ctrl + c 終止執行，然後按 command + q 退出終端機。
  2. 請在 VSCode 編輯器的界面按 command + w 關閉所有的標籤頁，然後點擊任務欄左上角 VSCode > Quit 退出編輯器。
  3. 在開始第二遍，第三遍作業時，同樣執行以上兩個步驟。

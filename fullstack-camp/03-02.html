# 2-2 建立新專案

# 目標

  • 這一節的目標，是要帶領大家建立一個全新的 Rails 空專案。

# 步驟

# Step 1. 確認 Ruby 版本

輸入 ruby -v

確認一下現在的 Ruby 版本。這份教材用的 Ruby 版本是 ruby-2.7.1

# Step 2. 安裝 Rails （版本 6.0.3.2 或以上)

輸入 rails -v

確認一下現在的 Rails 版本。這份教材用的 Rails 版本是 6.0.3.2。

# Step 3. 建立一個新 Rails 項目 （版本 6.0.3.2)

輸入 rails new rails101
（做第二遍的同學，請輸入 rails new rails102）
（做第三遍的同學，請輸入 rails new rails103）

# Step 4. 進入新建好的項目, 並建立 git 做版本控制

請跟著指南，輸入每一行指令。

  • cd rails101
    （做第二遍的同學，請輸入 cd rails102）
    （做第三遍的同學，請輸入 cd rails103）

  • git add .

  • git commit -m "Initial Commit"

# 2-3 幫 Rails 專案穿上衣服“Bootstrap”

# 目標

本章的作業目標，是要帶領各位把目前的空專案，穿上“衣服”，讓各位在做這個程式練習時，不再感到專案是在“裸奔”！

# 步驟

# Step 0. git checkout -b ch01

這裡我們要來示範一個 git 功能，輸入：

git checkout -b ch01

等一下在這一章做的動作，我們 commit 進去的程式碼都會屬於 ch01 這個分支。

# Step 1. 使用 yarn 下載 bootstrap 必要的前端套件

Bootstrap 是 Web 設計圈有名的 CSS 框架，提供了開發項目快速便利的元件素材，如按鈕、下拉選單、表格等等樣式。

在 Rails 上要幫專案穿上 Bootstrap，可以直接使用前端的 yarn 來安裝套件。

在終端機輸入

yarn add bootstrap jquery popper.js

# Step 2. 將 Bootstrap 的相關套件 jquery、popper.js 設定為全域使用

打開 config/webpack/environment.js ，加入以下程式碼

const { environment } = require('@rails/webpacker')

const webpack = require('webpack')
environment.plugins.append(
  'Provide',
  new webpack.ProvidePlugin({
    $: 'jquery',
    jQuery: 'jquery',
    Popper: ['popper.js', 'default'],
  })
)

module.exports = environment

# Step 3. 將 Bootstrap 的 CSS 套件裝進專案裡面

將 app/assets/stylesheets/application.css 更名為 application.scss

mv app/assets/stylesheets/application.css app/assets/stylesheets/application.scss

然後打開 app/assets/stylesheets/application.scss，加入以下內容：

@import 'bootstrap/scss/bootstrap'

# Step 4. 將 Bootstrap 的 Javascript 套件裝進專案裡面

然後打開 app/javascript/packs/application.js，加入以下內容：

require('bootstrap')

# Step 4. 將變更 commit 進去 git 裡面

git add .  
git commit -m "add bootstrap to project"

# 2-5 製作漂亮的“提示訊息”

# 目標

  • 你應該看過一些網站，有這樣的提示訊息吧？
  • 這一節的目的在於讓你學習到，在 Rails 內如何配合 Bootstrap 內建的特效，建立“提示訊息”。

# 步驟

# Step 1. 新增 app/views/common/_flashes.html.erb

touch app/views/common/_flashes.html.erb

填入

# app/views/common/_flashes.html.erb
<% if flash.any? %>
  <% user_facing_flashes.each do |key, value| %>
    <div class="alert alert-<%= flash_class(key) %> alert-dismissible fade show" role="alert">
      <%= value %>
      <button type="button" class="close" data-dismiss="alert" aria-label="Close">
        <span aria-hidden="true">&times;</span>
      </button>
    </div>
  <% end %>
<% end %>

# Step 2. 加入 app/helpers/flashes_helper.rb

touch app/helpers/flashes_helper.rb

加入以下內容：

# app/helpers/flashes_helper.rb
module FlashesHelper
  FLASH_CLASSES = { alert: "danger", notice: "success", warning: "warning"}.freeze

  def flash_class(key)
    FLASH_CLASSES.fetch key.to_sym, key
  end

  def user_facing_flashes
    flash.to_hash.slice "alert", "notice", "warning"
  end
end

# Step 3. 在 application.html.erb 內加入 flash 這個 partial

在 <%= yield %> 前加入 <%= render "common/flashes" %>

<%= render "common/flashes" %>
<%= yield %>

# Step 5. git 存檔

git add .  
git commit -m "add bootstrap flash function"

# Step 6: 測試 flash helper 的功能

修改 app/controllers/welcome_controller.rb。加入 flash[:notice] = "早安！你好！"。你應該可以看到系統跳出“綠色”提示窗。

# app/controllers/welcome_controller.rb
classWelcomeController < ApplicationController
  def index
    flash[:notice] = "早安！你好！"
  end
end

修改 app/controllers/welcome_controller.rb。加入 flash[:alert] = "晚安！該睡了！"。你應該可以看到系統跳出“紅色”提示窗。

# app/controllers/welcome_controller.rb
classWelcomeController < ApplicationController
  def index
    flash[:alert] = "晚安！該睡了！"
  end
end

修改 app/controllers/welcome_controller.rb。加入 flash[:warning] = "這是 warning 訊息！"。你應該可以看到系統跳出“黃色”提示窗。

# app/controllers/welcome_controller.rb
classWelcomeController < ApplicationController
  def index
    flash[:warning] = "這是 warning 訊息！"
  end
end

本節常見問題

我看不懂 FlashesHelper 那邊，可以解說一下嗎？

  • 我們後面會解釋，請先跳過。
  • 你只要先記得三件事：
    • flash[:notice] 是綠色訊息
    • flash[:alert] 是紅色訊息
    • flash[:warning] 是黃色訊息

# 2-6 參考：git 常用指令教學

這邊是一些常用的 git 指令，常用就會自然記起來，不用硬背，你可以在遇到問題的時候再回來看這篇。

# 存檔

最一開始的時候要先執行存檔：

cd 你的專案文件夾名稱
git add .
git commit -m "對於修改部分的簡單敘述"

# 新增一個分支

git checkout -b 你想取的分支名稱

這樣你的存檔會自動被保留在這個分支裡。
如果不小心做爛了，可以直接切回上一個分支去做。

# 切換分支

git checkout 你想要切換到的分支名稱

舉例來說，假設你從01開始做，做到 02 的時候爛掉了
你可以 git checkout 01 回到 01 再重新開始做

那這個爛掉的 02 要怎麼辦呢？雖然你可以產生另一個新的 02-1 ，但還是建議把錯誤的刪除掉，這樣才不會做到最後變得亂七八糟的。

# 刪除掉 branch 的方法

先離開你要刪除的分支，接著執行下列指令即可

git branch -D 你要刪除的分支名稱
